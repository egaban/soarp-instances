# Stochastic Orienteering Arc Routing Problem Instances

This directory contains instances of the Stochastic Orienteering Arc Routing
Problem (SOARP), aplied to dengue cases on street networks.

## Problem description

We assume a strong correlation between dengue case counts and mosquito
populations. The goal is to select a route (tour) that maximizes the total
number of dengue cases covered (prize), subject to a time budget.

Each instance provides observed (present) case counts and multiple future
scenarios generated by a multi‑agent simulator.

## Directory structure

The instances are organized into folders according to the number of future
scenarios per instance:

- `03-scenarios/`: 3 future scenarios
- `05-scenarios/`: 5 future scenarios
- `09-scenarios/`: 9 future scenarios
- `50-scenarios/`: 50 future scenarios

Inside each folder, individual instance files are stored in JSON format.

## Instance file format

Each JSON instance file has the following top‑level fields:

```
{
  "graph": {
    "nodes": [
      { "id": "0", "latitude": ..., "longitude": ... },
      ...
    ],
    "edges": [
      {
        "source": "0",
        "target": "1",
        "time": 12.34,
        "profits": [p0, p1, ..., pN]
      },
      ...
    ]
  },
  "max_time": T,
  "distributions": [w1, w2, ..., wN],
  "instance_type": "Stochastic"
}
```

- **graph.nodes**: list of nodes with unique IDs and (latitude, longitude).
- **graph.edges**: list of edges (directed streets). Each edge has:
  - `time`: travel time along the edge.
  - `profits`: an array of size `N+1`, where:
    - `profits[0]` = observed (present) dengue cases on that street.
    - `profits[i]` = simulated cases in future scenario _i_, for _i_ = 1..N.
- **max_time**: total time budget for the tour.
- **distributions**: probability distribution over the N future scenarios (sums
  to 1).
- **instance_type**: `"Stochastic"`.

## File naming convention

```
<network>-<|V|>-<|E|>-<instance_index>[-<N>scenarios].json
```

- `<network>`: network abbreviation (e.g., `as`, `ln`).
- `<|V|>`: number of nodes.
- `<|E|>`: number of edges.
- `<instance_index>`: index of the random seed/instance (1..6).
- `-<N>scenarios` (only in `03-scenarios`, `05-scenarios`, `09-scenarios`):
  number of future scenarios.

In the `50-scenarios` directory, filenames omit the `-50scenarios` suffix (since
it is implied by the directory).
